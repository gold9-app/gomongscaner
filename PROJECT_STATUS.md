# 고몽스캐너 개발 기록

최종 업데이트: 2026-04-24

## 1. 프로젝트 개요

- 앱명: `고몽스캐너`
- 목적: 포켓몬 카드의 현재 시세를 빠르게 확인하는 PWA 앱
- 핵심 사용 흐름:
  1. 카드 사진 업로드 또는 카드명/번호 검색
  2. AI가 카드 identity 정리
  3. KREAM, SNKRDUNK, PriceCharting, eBay 기준으로 시세 추정
  4. 결과를 앱형 UI로 표시
  5. 보관함에 저장 후 다시 조회

현재 방향은 "폼 기반 웹페이지"가 아니라 "모바일 앱처럼 바로 스캔하고 결과를 보는 경험"에 맞춰 조정 중이다.

## 2. 현재 기술 스택

- Next.js 16
- React 19
- TypeScript
- `app` router
- 주요 AI/검색 연동:
  - Gemini: 이미지/텍스트 기반 카드 인식
  - Claude: 보조 추론
  - Perplexity: 시세 리서치

## 4. 지금까지 구현된 핵심 기능

### 4-1. 카드 시세 조회 API

- `/api/estimate` 구현 완료
- 입력 모드:
  - 사진 업로드
  - 텍스트 검색
- 카드 타입:
  - `single`
  - `psa`
- 시세 근거 수집 소스:
  - KREAM
  - SNKRDUNK
  - PriceCharting
  - eBay sold/current

### 4-2. 카드 인식 정확도 개선

- 기존에는 일부 카드 테스트 케이스 위주 보정이 섞여 있었음
- 현재는 특정 카드 하드코딩이 아니라 일반화된 구조로 변경
- 지원 방향:
  - 일반 TCG 카드
  - PSA/BGS/CGC 슬랩 카드
  - 라벨이 없는 이미지
  - 비TCG 포켓몬 카드/굿즈 계열
    - Old Maid
    - Carddass
    - Topsun
    - Bandai
    - Meiji
    - Sticker
- OCR/이미지 evidence 기반으로:
  - 카드명
  - 언어
  - 세트/상품군
  - 카드 번호
  - 등급사
  - 등급
  - 인증번호
  를 정리한 뒤 검색어를 확장하도록 구성

### 4-3. 검색어 정규화 방향

- 사용자 자유 입력은 그대로 받되 내부 검색은 더 단순하고 강한 키워드 중심으로 만듦
- 예:
  - 사용자가 길게 입력해도 내부에서는 카드명, 번호, 상품군, 등급 중심으로 재구성
- 검색 결과 매칭도 특정 카드명 하드코딩이 아니라 후보 이름군 기반으로 비교하도록 수정

### 4-4. 프론트 UI

- 상단:
  - 좌측 햄버거 메뉴
  - 우측 실시간/테스트 상태 표시
- 메인:
  - 홈은 스캐너 대기 화면 중심
  - 검색 모드는 별도 입력 시트 느낌으로 분리
- 하단 네비:
  - 촬영
  - 홈
  - 검색
- 메뉴:
  - 홈
  - 검색
  - 보관함
  - 최근 검색
- 보관함:
  - localStorage 저장
  - 다시 조회
  - 삭제
  - 저장된 결과 다시 열기

### 4-5. 앱 느낌 개선

최근 변경으로 아래 흐름을 반영했다.

- 기본 홈 검색어 제거
- 첫 화면을 스캐너 대기 화면으로 정리
- 문구를 짧고 자연스럽게 정리
- 검색 중 단계 UI 추가
  - 카드 인식
  - 검색어 정리
  - 거래가 확인
  - 시세 계산
- 시세 결과는 이제 스캐너 옆 패널이 아니라 별도 결과 화면처럼 전환
- 결과 화면에서:
  - 보관하기
  - 다시 스캔
  이 가능

## 5. 현재 UX 상태 요약

지금은 "기능은 동작하고, 앱 구조도 잡혔지만 아직 완성형 모바일 앱의 마감은 부족한 상태"다.

좋아진 점:

- 홈이 앱처럼 단순해짐
- 결과 화면 분리로 웹툴 느낌이 줄어듦
- 검색 진행 단계가 보여서 AI 앱 느낌이 생김

아직 부족한 점:

- 결과 화면 전환 애니메이션이 약함
- 카드 썸네일/실제 카드 비주얼이 부족함
- 보관함이 아직 리스트 중심이라 컬렉션 느낌이 약함
- PWA 설치 경험과 아이콘 마감이 아직 부족함
- 텍스트/컴포넌트 밀도가 아직 조금 높은 편
